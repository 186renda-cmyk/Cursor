素材1:
Cursor 是一个 AI 驱动的代码编辑器，它能帮你写代码、改代码、甚至理解代码。如果你刚接触，可能会觉得界面有点像黑客电影，但其实用起来非常简单直接。
快速入门：从零到做出第一个程序
想快速上手，可以跟着这几个步骤来：
		​​下载与汉化​​：去官网下载安装包，安装后打开。默认是英文界面，可以按快捷键 Cmd/Ctrl + Shift + P 打开命令面板，输入“Chinese”安装中文语言包。
		​​安装环境​​：同样在命令面板里，输入“Python”并安装，这样就能开始写 Python 程序了。
		​​开始对话​​：按快捷键 Cmd/Ctrl + I ，一个对话框就会出现在右侧。
		​​提出需求​​：在对话框里直接用中文描述你的需求，比如“帮我开发一个 GUI 程序，实现全网 AI 新闻相关知识的获取”。
		​​查看与运行​​：AI 会开始生成代码，你看到提示后点击“Accept”或按快捷键 Cmd/Ctrl + Enter 。代码生成后，它会告诉你如何运行。在左侧项目文件夹里找到生成的程序文件，右键选择“在终端中运行”即可。
核心功能与使用技巧
Cursor 的强大之处在于它不仅仅是写代码，还能帮你做很多事。
		​​三大核心模式​​
		​​Ask 模式​​：适合快速提问，比如“这段代码是干嘛的？”。选中代码按 Cmd/Ctrl + K ，再按 Alt + Enter ，AI 会直接在编辑器里回答你。
		​​Edit 模式​​：专为长代码优化，适合直接在文件中生成或修改代码，比如“帮我写个上传函数”。
		​​Agent 模式​​：交互式助手，适合复杂任务和持续对话，能跨文件编辑和执行命令。
		​​高效快捷键​​
		​​Tab​​：自动补全代码。
		​​Cmd/Ctrl + K​​：内联编辑，选中代码后按此键，告诉 AI 具体要怎么修改，比如“加个异常判断”。
		​​Cmd/Ctrl + I​​：打开 Agent 模式的对话面板。
		​​Cmd/Ctrl + L​​：打开 Chat 面板，直接对话生成代码。
		​​@提示符​​：这是 Cursor 的隐藏神器，用它可以快速引入上下文。
		​​@notepad​​：引用你写好的需求文档，让 AI 不跑偏。
		​​@web​​：直接抓取网页内容作为参考。
		​​@files/@folders​​：引用特定文件或整个文件夹的代码，让 AI 更懂你的项目。
避坑与进阶技巧
		​​免费版限制​​：免费账户请求次数有限，且只能使用基础模型。如果想解锁全部功能和更强大的模型，需要订阅 Pro 版。
		​​Review 再 Apply​​：AI 生成的代码一定要自己 Review 一遍再应用，避免引入隐藏的 bug。
		​​分步构建​​：不要一次性让 AI 构建整个复杂项目，容易出错。最好分模块、分功能逐步实现。
		​​善用 MCP​​：通过设置页面连接 MCP（Model Connection Protocol） 服务，比如 Being 中文搜索 或 高德地图 ，可以让 AI 调用外部工具，获取实时数据，变得更智能。
		​​切换模型​​：不同模型擅长的领域不同。比如写代码可以试试 Gemini-2.5-pro ，学习新知识可以用 GPT-4o-mini 。如果对结果不满意，换一个模型试试。

素材2:
Cursor AI是一款强大的AI编程助手，以下是其主要使用方法：

安装与登录

访问Cursor官网（https://cursor.sh/），下载对应系统的安装包并完成安装。
首次启动时，需使用邮箱、Google或GitHub账号登录。


基础操作

打开项目：点击左上角“文件”→“打开文件夹”，选择本地项目文件夹；或通过“克隆仓库”功能从Git远程仓库拉取项目。
代码编辑：在代码编辑区直接输入代码，Cursor会自动提供代码补全建议，按Tab键可接受建议。


AI核心功能

代码补全：输入代码时，Cursor会根据上下文自动补全代码片段。例如输入函数名或变量名，它会提示可能的实现代码。
AI聊天（Chat）：按Ctrl+L（Windows/Linux）或Cmd+L（Mac）打开聊天窗口，用自然语言描述需求，如“生成一个计算两个数之和的函数”“解释这段代码的作用”等，Cursor会生成代码或提供解释。
内联编辑：选中代码后按Ctrl+K（Windows/Linux）或Cmd+K（Mac），输入修改要求，Cursor会生成修改后的代码，可选择接受或拒绝。
Composer模式：按Ctrl+I（Windows/Linux）或Cmd+I（Mac）打开Composer窗口，用于多文件编辑，如创建新功能模块、跨文件重构等。
Plan Mode：按Ctrl+Shift+P（Windows/Linux）或Cmd+Shift+P（Mac），输入“Plan”打开Plan Mode，可可视化规划大型任务，生成实施计划。
Debug Mode：在设置中启用Debug Mode，运行应用时，Cursor会自动收集运行时日志，分析并定位问题，提供修复方案。


上下文控制

引用代码或文件：在聊天或Composer中，使用@符号引用特定代码、文件或文件夹，如@code LocalAdmAgent.ts、@file initZZERCoreStore.ts，帮助Cursor更精准地理解上下文。
配置项目规则：在项目根目录创建.cursor/rules文件夹，添加规则文件（如.mdc），定义代码规范、技术栈等，确保AI生成的代码符合项目要求。


多代理系统（2.0+）

打开Agent侧边栏，添加多个代理，分配不同任务（如分析代码、生成文档、编写测试等），Cursor会并行处理并评估结果，推荐最佳方案。



使用Cursor时，建议从简单功能开始尝试，逐步熟悉其能力。AI生成的代码需人工审核，确保质量和安全性。
